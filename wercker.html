Play framework continuous integration with Wercker.

Most developer does not think about time that they spend on testing/deployment process. I hope that after reading of this article, you will have more free time for coding, not for deploy. This information will be useful for developers that works on projects with small iterations and frequent releases.

What is continuous integration?

Continuous Integration is a software development practice where members of a team integrate their work frequently, usually each person integrates at least daily - leading to multiple integrations per day. Each integration is verified by an automated build (including test) to detect integration errors as quickly as possible. Many teams find that this approach leads to significantly reduced integration problems and allows a team to develop cohesive software more rapidly.

Choosing of CI tools

During choosing of CI I have some requirements as cloud computing without any local installations, so typical decisions as TeamCity or Jenkins have been omitted. Also I want to use this tool on my personal projects that stored on private GitHub/Bitbucker repositories, so pricing of service was been important. As a result I choosed Wercker becouse it was free for personal use on private repositories, have simple configuration based on docker-like boxes and easily integrates into existing projects.

Usage of Wercker

After registering of account, you must create a inner application that will integrate with you Git repository (GitHub and Bitbucket integraion is provided). Wercker will create wercker.yml file in root of your repository, this file must contains your integration script. After each push into attached repository, Wercker will compose docker box based on you yml file and execute actions in pipeline. Also you can add variables into you pipeline from application settings page and use it in your script as $VARIABLE. For easier integration, you can use boxes registry with pre-installed software that will be necessary for project.

Integration of Play2 application

For integrating of Play application we must have pre-installed JDK and Activator. If your dont wanna write install script himself, you can choose one of already created. There are multiply versions for different JDK: Oracle JDK 8, OpenJDK 7, OpenJDK 6. For current example I used OpenJDK 7, but you can easily change it by box parameter.

Application build will have three easy steps, as compile, testing and build of application archive. After executing of every pipeline action, Wercker will check exit code and continue process if result would been true.

Deploy on server

Wercker provides integration with Heroku and OpenShift. But you can deploy application throught SSH without any problems. Anyway you must add deploy target in Wercker settings that will contains auto deploy settings and inner variables. Also if your use Digital Ocean, there are nice guide written by Wercker team.

For my project I used script provided in Wercker blog for DO with some improvements. For every deploy I load created archive throught SSH, unzip it and run application in screen.
